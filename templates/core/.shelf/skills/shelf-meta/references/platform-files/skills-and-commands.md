# Skills, Commands, Prompts, And Workflows

Skills and commands are textual entry points for user interaction with AgentOS Shelf. Codex and Claude Code use different platform surfaces, but their core purpose is the same: tell the AI how to enter the AgentOS Shelf flow when the user expresses a certain intent.

## Conceptual Differences

| Type | Trigger mode | Best for |
| --- | --- | --- |
| skill | AI auto-match or explicit user mention | Long-term capabilities, workflow rules, modification guides. |
| command | Explicit user invocation | Clear operation entry points such as continue and finish-work. |
| workflow | Explicit user selection or platform auto-match | Future platform adapter shape; not generated by the current CLI. |

AgentOS Shelf workflow skills usually share one semantic set: brainstorm, before-dev, check, update-spec, break-loop. Multi-file built-in skills such as `shelf-meta` use layered references.

## Common Paths

| Platform | Common entries |
| --- | --- |
| Claude Code | `.claude/skills/`, `.claude/commands/` |
| Codex | `.agents/skills/`, `.codex/agents/`, `.codex/hooks/` |

In a user project, use the files actually generated by init as authoritative.

## Skill Structure

A common skill is a directory:

```text
shelf-meta/
├── SKILL.md
└── references/
```

`SKILL.md` should tell the AI:

- When to use this skill.
- Which reference to read first for the current task.
- What not to do.

References hold longer explanations so the entry file does not contain everything.

## Command/Workflow Structure

Commands and workflows are usually single files. Their content should include:

- When to use it.
- Which `.shelf/` files to read.
- Which scripts to run.
- How to report after completion.

They should not store task state; task state belongs in `.shelf/tasks/` and `.shelf/.runtime/`.

## Local Change Scenarios

| User need | Edit location |
| --- | --- |
| Change AI auto-trigger rules | The corresponding skill's frontmatter description. |
| Change user command behavior | The corresponding common command or platform command file. |
| Add a project-local skill | Platform skill directory, or shared `.agents/skills/`. |
| Let multiple platforms share one capability | Use `.agents/skills/` for Codex and `.claude/skills/` for Claude Code. |
| Change finish/continue entry points | `.shelf/templates/common-commands/` and the generated Claude command / Codex skill outputs. |

## Modification Principles

1. **Keep entry files short; references carry long content**. This matters especially for multi-file skills like `shelf-meta`.
2. **Make trigger descriptions specific**. A description that is too broad can mis-trigger; one that is too narrow may not trigger.
3. **Keep the same semantics consistent across Codex and Claude Code**. File formats can differ, but behavior descriptions should match because they come from the same common source.
4. **Put project-specific capabilities in local skills**. Do not put team-private flows into public `shelf-meta`.

If the user only wants local AI to know one more project rule, usually create a project-local skill or update `.shelf/spec/` instead of changing a AgentOS Shelf built-in workflow skill.
